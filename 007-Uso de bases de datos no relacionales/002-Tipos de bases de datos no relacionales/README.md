Iintroducción
Este documento explica el uso de MongoDB como base de datos NoSQL para la gestión de una colección llamada facturas, mostrando cómo realizar operaciones básicas como insertar, consultar, actualizar y eliminar datos. Estas operaciones se implementan en distintos lenguajes de programación, lo que permite entender su funcionamiento en entornos reales de desarrollo.

Aspectos tecnicos
A nivel técnico, se utiliza MongoDB como sistema de base de datos, junto con tres entornos principales: PHP mediante el driver de MongoDB, la consola de MongoDB  y Python usando PyMongo. Todos ellos trabajan sobre la misma base de datos empresa2026 y la colección facturas, permitiendo manipular documentos en formato JSON de manera flexible.

Para que sirve
Este sistema sirve para gestionar información de facturación o clientes en aplicaciones empresariales, facilitando el almacenamiento y consulta de datos sin una estructura rígida. Es especialmente útil en aplicaciones web, APIs y sistemas que requieren escalabilidad y rapidez en el acceso a la información.

Conclusion
En conclusión, MongoDB ofrece una forma eficiente y moderna de trabajar con datos mediante operaciones CRUD accesibles desde distintos lenguajes. Su flexibilidad y facilidad de integración lo convierten en una herramienta adecuada para proyectos actuales de desarrollo backend y gestión de información dinámica.
